Modelagem de Dados

Dados: São fatos brutos, que podem ser números, textos ou imagens, que por si só não transmitem uma mensagem completa. Eles são a matéria-prima utilizada para criar informações. Por exemplo, “23”, “azul” e “Maria” são dados.

Informações: São dados processados ou organizados de uma maneira que os torna significativos ou úteis. Quando os dados são colocados em um contexto ou interpretados de alguma forma, eles se transformam em informações. Usando o exemplo anterior, "Maria tem 23 anos e gosta da cor azul" é uma informação.

  • Organização: Permite o armazenamento estruturado de dados.
  • Eficiência: Facilita a recuperação e atualização dos dados.
  • Integridade: Mantém a consistência e a integridade dos dados.
  • Segurança: Fornece mecanismos para proteger os dados contra acessos não autorizados.
  • Centralização: Permite o armazenamento centralizado dos dados, o que facilita o compartilhamento entre diferentes usuários ou sistemas.

  • Antes dos bancos de dados modernos, o gerenciamento de dados era feito usando sistemas de arquivos. Cada aplicação tinha seu próprio sistema de arquivos, o que muitas vezes levava à redundância e inconsistência dos dados.
  • A recuperação e a atualização dos dados eram mais lentas e propensas a erros.
  • Não havia mecanismos de controle de concorrência robustos, o que poderia levar a problemas de integridade dos dados.
  • A segurança e a integridade dos dados eram muitas vezes dependentes das capacidades do sistema operacional e não eram tão granulares ou flexíveis quanto em um BD.

Um banco de dados é um conjunto organizado de dados, armazenados de forma estruturada, para serem acessados, gerenciados e atualizados. Eles podem ser classificados de acordo com o seu conteúdo, como bibliográfico, texto, numérico ou imagens. Bancos de dados são geralmente gerenciados por um Sistema de Gerenciamento de Banco de Dados (SGBD).

  • Oracle Database: Um SGBD relacional comercial muito popular e robusto.
  • MySQL: Um SGBD relacional open-source amplamente usado, agora de propriedade da Oracle Corporation.
  • Microsoft SQL Server: SGBD relacional desenvolvido pela Microsoft.
  • PostgreSQL: Um SGBD relacional open-source poderoso e extensível.
  • MongoDB: Um banco de dados NoSQL orientado a documentos.

ACID é um conjunto de propriedades que garantem que as transações em um banco de dados sejam processadas de forma confiável. As propriedades são:

  • Atomicidade (Atomicity): Garante que todas as operações dentro de uma transação sejam concluídas com sucesso; caso contrário, a transação é abortada.
  • Consistência (Consistency): Garante que uma transação leve o banco de dados de um estado válido para outro.
  • Isolamento (Isolation): Garante que as transações sejam executadas de forma isolada, sem interferências de outras transações.
  • Durabilidade (Durability): Uma vez que uma transação é concluída, ela é garantida de ser permanente, mesmo em caso de falhas.
